# Qiita 記事投稿ガイドライン

## 📋 このガイドラインについて

このガイドラインは、Shiftize プロジェクトの学習記録を Qiita で公開する際の指針です。  
大学生がポートフォリオとして活用するための、質の高い記事を書くためのガイドです。

---

## 🎯 記事の目的

1. **学習記録**: 自分が学んだことを記録する
2. **知識の共有**: 他の人にも役立つ情報を提供する
3. **ポートフォリオ**: 就職活動で使える実績を作る
4. **フィードバック**: コメントでアドバイスをもらう

---

## 📝 記事の種類

### 1. 学習ロードマップ
- **目的**: 学習計画を共有する
- **対象読者**: 同じように学習している人
- **テンプレート**: `QIITA_ROADMAP_TEMPLATE.md`

### 2. 開発日記
- **目的**: 毎日の学習記録を残す
- **対象読者**: 同じように開発している人
- **テンプレート**: `QIITA_DAILY_LOG_TEMPLATE.md`

### 3. 技術解説記事
- **目的**: 特定の技術について詳しく解説する
- **対象読者**: その技術を学びたい人
- **例**: 「React Hooks の useEffect を完全理解する」

### 4. 問題解決記事
- **目的**: 遭遇した問題と解決方法を共有する
- **対象読者**: 同じ問題に遭遇した人
- **例**: 「Firebase Authentication でエラーが出た時の対処法」

---

## ✍️ 記事の書き方

### タイトルの付け方

**良い例**:
- `【学習記録】React + TypeScript でシフト管理アプリ開発 - Week 1`
- `【解決方法】Firebase Firestore のリアルタイム更新が動かない`
- `【初心者向け】React Hooks の useEffect を完全理解する`

**悪い例**:
- `日記`（内容が分からない）
- `エラー`（何のエラーか分からない）
- `React`（範囲が広すぎる）

### 記事の構成

#### 1. 導入部分
```markdown
## 📋 この記事について

- **対象読者**: [誰向けか]
- **前提知識**: [必要な知識]
- **所要時間**: [読むのに必要な時間]
```

#### 2. 本文
- **見出しを適切に使う**（##, ###）
- **コードブロックを使う**
- **画像や図を入れる**（必要に応じて）
- **箇条書きを使う**

#### 3. まとめ
```markdown
## 🎯 まとめ

- ポイント1
- ポイント2
- ポイント3
```

### コードの書き方

**良い例**:
````markdown
```typescript
// コメントで説明を追加
const MyComponent = () => {
  const [count, setCount] = useState(0);
  
  return (
    <div>
      <p>カウント: {count}</p>
      <button onClick={() => setCount(count + 1)}>増やす</button>
    </div>
  );
};
```
````

**悪い例**:
````markdown
```typescript
const MyComponent = () => {
  const [count, setCount] = useState(0);
  return <div><p>{count}</p><button onClick={() => setCount(count + 1)}>+</button></div>;
};
```
````

### 画像の使い方

- **スクリーンショット**: 実際の動作を説明する
- **図解**: 概念を説明する
- **コードの実行結果**: 結果を視覚的に示す

**画像のアップロード方法**:
1. Qiita のエディタで画像をドラッグ&ドロップ
2. または、GitHub に画像をアップロードして URL を貼り付け

## 🏷️ タグの付け方

### 必須タグ
- `React`（または `Vue.js`, `Angular` など）
- `TypeScript`（または `JavaScript`）
- `Firebase`（使用している場合）

### 推奨タグ
- `学習記録`（学習ロードマップや日記の場合）
- `初心者向け`（初心者向けの記事の場合）
- `大学生エンジニア`（大学生が書いた記事の場合）
- `ポートフォリオ`（ポートフォリオとして公開する場合）

### タグの例
```
#React #TypeScript #Firebase #学習記録 #大学生エンジニア #ポートフォリオ
```

## 📊 記事の品質を上げるコツ

### 1. 読者目線で書く
- **「自分が知りたいこと」を書く**
- **「なぜそうなるのか」を説明する**
- **「実例を交える」**

### 2. 構造を明確にする
- **見出しを適切に使う**
- **箇条書きを使う**
- **コードと説明を分ける**

### 3. 視覚的に分かりやすく
- **絵文字を使う**（適度に）
- **コードブロックを使う**
- **画像を入れる**

### 4. 継続的に更新する
- **コメントへの返信**
- **記事の更新**（情報が古くなった場合）
- **関連記事へのリンク追加**

---

## 🚫 避けるべきこと

### 1. コピペ記事
- **他人の記事をそのままコピーしない**
- **参考にした場合は出典を明記する**

### 2. 情報が古い記事
- **最新の情報を確認する**
- **バージョン情報を記載する**

### 3. 説明が不十分
- **「分かっている前提」で書かない**
- **専門用語には説明を追加する**

### 4. コードだけの記事
- **コードには説明を追加する**
- **なぜそのコードなのか説明する**
